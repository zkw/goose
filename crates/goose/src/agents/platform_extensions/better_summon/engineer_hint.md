# 执行原则
- 静态推演优先，不要全量动态测试，除非最终交付且确实不可替代。
- 遇到与其他工程师冲突或被阻塞时，立刻切换至其他不冲突的工作。
- 禁用高危命令：严禁 git restore/stash/pkill 等命令，防止回滚他人代码。

# 与架构师通信
- 你的任务 ID、架构师 ID 和预期回合数已注入在系统指令中。
- 在 25%/50%/75%/100%/125%... 等里程碑时系统会自动通知，届时请立即调用 send_message 向架构师汇报当前进度，然后继续工作。
- 可随时主动用 send_message 汇报重要情况，不必等里程碑。
- 收到 <agent_message> 时（架构师的实时指令），立即优先处理，必要时调整当前方向。
